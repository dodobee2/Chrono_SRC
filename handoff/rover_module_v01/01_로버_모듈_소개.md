# 01. 로버 모듈 소개

## 이 모듈이 하는 일

한 줄로 말하면 "스펙 숫자를 넣으면 시뮬레이션 차량이 나오는 기계"입니다.
도희님이 정한 RoverSpec 형식의 yaml 파일에 질량, 휠 크기, 축간거리 같은
숫자를 적으면, 코드가 그 숫자대로 Chrono 안에 로버를 조립합니다. 나중에
CAD와 구동계 부품이 확정되면 yaml 숫자만 바꾸면 되고, 코드는 손댈 필요가
없습니다.

scout와 main이 스펙 파일 두 개로 존재하는 이유도 여기에 있습니다. 우리
주제가 "정찰로버 데이터로 본 로버를 예측"하는 것이라서, 질량과 휠이 다른
두 차량을 같은 코드로 돌릴 수 있어야 하기 때문입니다.

## 지금 버전이 할 수 있는 것

대회 임무 기준으로 필요한 최소 기능부터 넣었습니다.

- 직진 주행: 네 바퀴 동일 속도(또는 토크) 명령
- 방향 전환: 좌우 바퀴에 서로 다른 명령을 주는 skid steer 방식.
  좌우를 역회전시키면 제자리 선회가 됩니다 (검증 결과 scout 기준
  4초에 222도 회전)
- 경사 등판: 오르막에서 필요한 토크가 모터 한계 안인지 확인
- 수납부피 검사: 공고문 한도(300×300×200 mm) 안에 들어가는지 자동 확인
- 임무 장비 표현: 카메라/안테나 마스트, 전방 depth 센서, 그루저 휠.
  이 요소들은 눈에 보이는 형상이고, 물리 계산은 몸체 박스와 원통 바퀴가
  담당합니다 (이유는 02 문서에 적었습니다)

## 산출물 목록

코드 원본은 `code/` 폴더에 그대로 넣었습니다. 아래 경로는 그 안 기준입니다.

### specs/scout_v01.yaml, specs/main_v01.yaml — 로버 스펙 파일

로버 한 대의 사양을 적는 파일입니다. 항목은 도희님이 카톡에 올린 RoverSpec
dataclass와 완전히 같습니다. 지금 들어 있는 숫자는 전부 제가 넣은 초기
추정치라서 파일 안에 TODO 주석을 달아뒀습니다. 확정되는 대로 갱신하겠습니다.

    scout_v01: 1.8 kg, 휠 반경 3 cm, 축간거리 16 cm, 4륜 구동, 속도 명령
    main_v01:  2.8 kg, 휠 반경 4.5 cm, 축간거리 22 cm, 4륜 구동

질량은 팀에서 정한 1.5~3 kg 범위 안에서 잡았습니다. 근거는 02 문서에
있습니다.

### src/rover_schema.py — 스펙 읽기와 검사

yaml을 읽어서 RoverSpec으로 바꾸고, 값이 말이 되는지 검사합니다. 질량이
음수라든가, 무게중심이 지상고보다 낮다든가, 필드가 빠졌다든가 하면
어디가 왜 틀렸는지 알려주며 멈춥니다. 잘못된 스펙이 조용히 시뮬레이션까지
흘러가는 사고를 막는 층입니다.

### src/rover_builder.py — 로버 조립

RoverSpec을 받아 Chrono 차량을 만듭니다. 물리 형상은 박스 몸체에 원통
바퀴 네 개이고, 그 위에 마스트·센서·그루저 같은 시각 요소가 붙습니다.
바퀴마다 모터가 있고 좌우 독립 명령(set_command_lr)으로 선회합니다.

조립이 끝나면 모델 전체 질량이 스펙의 mass_kg와 일치하는지 코드가
스스로 확인합니다. 어긋나면 그 자리에서 에러가 납니다.

한 가지 중요한 설계 결정이 있습니다. 빌더는 시뮬레이션 세계(system)를
직접 만들지 않고, 밖에서 만들어진 세계에 로버를 "추가"만 합니다. 종민님이
TerrainScenario로 지형 세계를 만들면 거기에 이 함수를 불러 로버를 얹는
식으로 바로 합쳐집니다.

### src/test_ground.py — 검증용 바닥

제 파트 검증에만 쓰는 평평한 바닥입니다. 기울기 옵션이 있어 경사 시험에도
씁니다. 실제 지형(자갈밭, 소금 입자 등)은 종민님 파트라서 여기서는 만들지
않았습니다.

### src/logger.py — 주행 기록

매 순간의 위치, 자세(pitch/roll/yaw), 바퀴 회전속도, 슬립률, 토크, 에너지,
좌우 명령값을 CSV 파일로 남깁니다. 나중에 실물 로버 로그와 같은 항목을
비교하게 될 텐데, 그때 컬럼 형식은 팀 공통 로그 규격에 맞출 예정입니다.

### scripts/run_rover_check.py — 자동 검증

로버가 물리적으로 말이 되게 움직이는지 검사 26개를 자동으로 돌립니다.
내용은 03 문서에 정리했습니다.

### scripts/view_rover.py, scripts/render_rover.py — 3D 화면

view_rover는 창을 띄워 로버가 실시간으로 달리는 걸 보여주고, render_rover는
보고용 이미지를 찍습니다. images 폴더의 로버 그림이 이걸로 만든 것입니다.

### tests/ — 코드 단위 테스트

스펙 검사, 질량 일치, 선회 방향, 수납부피 같은 26개 항목을 pytest로
확인합니다. 코드를 고칠 때마다 돌려서 기존 기능이 깨지지 않았는지 봅니다.

## 직접 돌려보고 싶으면

연습 레포 때 만든 conda 환경(chrono)을 그대로 씁니다. 새로 설치할 건
없습니다. code 폴더를 연습 레포와 같은 컴퓨터에 두고 실행하면 됩니다.

    conda activate chrono
    python scripts/run_rover_check.py     # 검증 전부 실행, 그래프 저장
    python scripts/view_rover.py          # 3D 창으로 주행 구경
    python -m pytest tests/ -q            # 코드 단위 테스트
