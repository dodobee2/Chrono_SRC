# 04. 용어 풀이

문서와 코드에 나오는 용어를 모았습니다. 회의 자료(신규 팀원 설명서 11장)의
용어표와 겹치는 건 같은 뜻으로 썼습니다.

## 로버 스펙(RoverSpec) 항목

| 항목 | 뜻 |
| --- | --- |
| rover_id | 로버 이름표. scout_v01처럼 "어느 차의 몇 번째 버전"인지 구분합니다 |
| mass_kg | 로버 전체 질량. 배터리, 보드, 바퀴까지 다 합친 값입니다 |
| cg_xyz_m | 무게중심 위치. 원점은 네 바퀴 접지점의 한가운데(지면 높이)로 잡았습니다 |
| wheel_count | 바퀴 개수 |
| driven_wheel_count | 그중 모터가 달린 바퀴 개수. 4개면 4륜 구동입니다 |
| wheel_radius_m | 바퀴 반지름 |
| wheel_width_m | 바퀴 폭 |
| wheelbase_m | 축간거리. 앞바퀴 축과 뒷바퀴 축 사이 거리입니다 |
| track_width_m | 윤거. 왼쪽 바퀴와 오른쪽 바퀴 중심 사이 거리입니다 |
| ground_clearance_m | 지상고. 바닥에서 몸체 밑면까지 높이. 이만한 돌은 배로 넘을 수 있습니다 |
| command_type | 바퀴에 내리는 명령 종류. 속도(wheel_speed) 또는 토크(wheel_torque) |
| max_wheel_torque_nm | 바퀴 하나가 낼 수 있는 최대 토크 |
| max_wheel_speed_radps | 바퀴 최대 회전속도 |

## 단위 감 잡기

- 토크는 N·m(뉴턴미터)인데 우리 로버는 값이 작아 mN·m(밀리뉴턴미터,
  1/1000 N·m)로 적습니다. 등판에 13 mN·m라면 0.013 N·m입니다.
- 바퀴 회전속도 rad/s에 반지름을 곱하면 차 속도가 됩니다.
  반경 0.03 m 바퀴가 12 rad/s로 돌면 차는 0.36 m/s로 갑니다.
- 각도는 문서에서는 도(deg), 코드 내부에서는 라디안을 씁니다.

## 시뮬레이션 용어

무게중심(CG)
: 질량이 한 점에 몰려 있다고 칠 수 있는 위치입니다. 높으면 경사에서
뒤집히기 쉽고, 앞뒤로 치우치면 바퀴마다 눌리는 힘이 달라집니다.

슬립률(slip ratio)
: 바퀴가 도는 만큼 실제로 나아가는지 나타내는 값입니다. 바퀴 속도
(회전속도 × 반지름)와 차체 속도의 차이를 바퀴 속도로 나눕니다. 0이면
미끄러짐 없이 구르는 것이고, 1에 가까우면 제자리에서 헛도는 것입니다.
모래에서 이 값이 커지고, 우리 프로젝트에서 지형 성질을 추정하는 핵심
재료입니다.

pitch / roll / yaw
: 차체가 기운 방향입니다. pitch는 앞뒤 기울기(오르막에서 코가 들림),
roll은 좌우 기울기, yaw는 방향 전환(좌회전, 우회전)입니다.

skid steer (스키드 조향)
: 바퀴를 꺾지 않고 좌우 바퀴 속도 차이로 방향을 바꾸는 방식입니다.
탱크나 청소 로봇이 도는 원리와 같습니다. 좌우를 서로 반대로 돌리면
제자리 선회가 됩니다. 조향 기구가 없어 가볍고 단순해서 소형 로버에
많이 씁니다.

그루저(grouser)
: 바퀴 표면에 붙은 돌기입니다. 모래나 무른 흙에서 미끄러짐을 줄여
줍니다. 지금 모델에서는 눈에 보이는 형상으로만 붙어 있고, 접촉 계산은
매끈한 원통이 담당합니다. 무른 지형(SCM) 단계에서 효과를 넣게 됩니다.

rigid 지형
: 아무리 눌러도 변형되지 않는 딱딱한 바닥입니다. 콘크리트라고 생각하면
됩니다. 반대로 SCM은 바퀴 자국이 남는 무른 토양 모델이고 다음 단계에서
씁니다.

NSC 접촉
: Chrono가 물체끼리 닿는 걸 계산하는 방식 중 하나입니다. 물체가 서로
파고들지 않는다는 조건을 수학적으로 강제합니다. 딱딱한 물체 여러 개가
부딪히고 구르는 상황에 알맞아서 채택했습니다.

timestep(시간 간격)
: 시뮬레이션이 세상을 갱신하는 간격입니다. 저희는 0.002초, 즉 1초를
500번 쪼개 계산합니다. 작을수록 정확하지만 느립니다.

에너지 proxy
: 각 바퀴의 토크 × 회전속도(= 기계적 출력)를 시간에 대해 쌓은 값입니다.
모터 효율, 회로 손실은 아직 없으므로 "실제 배터리 소모"가 아니라
"경로가 요구하는 일의 양"으로 읽어야 합니다. 경로끼리 비교하는 용도입니다.

headless
: 3D 창 없이 계산만 돌리는 모드입니다. 결과는 CSV와 그래프 파일로
남습니다. 수백 번 반복 실험을 돌릴 땐 창이 없어야 빠릅니다.

토크-속도 선형 맵
: DC 모터의 성질을 직선 하나로 근사한 것입니다. 멈춰 있을 때 힘이
최대이고 빨리 돌수록 힘이 줄어 무부하 속도에서 0이 됩니다. 모터
데이터시트의 스톨 토크와 무부하 속도만 있으면 그릴 수 있습니다.
